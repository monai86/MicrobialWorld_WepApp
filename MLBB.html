Slide 7 — Algorithm Core

🧮 calcProbability(group, answers)

for each organism in group:
  for each answered test:
    if match    → score += weight
    if variable → score += weight/3
    if mismatch → score -= weight
    if KEY test mismatch  → score -= weight × 2
    if HARD exclusion     → cap score at 5%

  probability = (score / maxScore) × 100

return organisms sorted by probability

Slide 8 — How Agentic IDE Works

🤖 Agentic IDE ≠ Autocomplete ธรรมดา

Autocomplete:          Agentic IDE (Windsurf):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Slide 11 — Strengths (ข้อดีของแอพ)

✅ จุดแข็งของ BOK BAC
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🎯 ด้านการใช้งาน:
- Interactive — ไม่ใช่แค่อ่าน textbook
- ฟรี & ไม่ต้องติดตั้ง — เปิด browser ได้เลย
- ใช้ได้ทุก device (responsive)
- บันทึก case ได้ — ทบทวนภายหลัง

🧮 ด้าน Algorithm:
- Weighted scoring — ให้น้ำหนักต่างกันตาม key test
- Hard exclusion — ตัดเชื้อที่เป็นไปไม่ได้ออกทันที
- ครอบคลุม 7 กลุ่มเชื้อ + 40+ species

🤝 ด้านการพัฒนา:
- สร้างด้วย AI ทั้งหมด — ประหยัดเวลามาก
- Single file (index.html) — deploy ง่าย

Slide 12 — Limitations

⚠️ ข้อจำกัดที่มีอยู่
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 ด้านข้อมูล:
- ข้อมูลเชื้อ — ป้อนด้วยมือ อาจมี error
- ไม่ครอบคลุม anaerobes, fungi, parasites
- ไม่มี antibiogram / susceptibility data

🧮 ด้าน Algorithm:
- Rule-based เท่านั้น — ไม่ใช่ ML
- Weight ตั้งด้วยมือ — อาจ bias
- ไม่รองรับผล "borderline" หรือ atypical strains

💾 ด้าน Technical:
- ข้อมูลเก็บใน LocalStorage — หายได้
- Single HTML file — scale ยาก
- ไม่มี real-time sync ระหว่าง users
- ไม่ผ่าน clinical validation จริง
